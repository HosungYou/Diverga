# Docs Language Policy

## Policy

Active LongTable engineering documents should be written in English.

This keeps package contracts, provider behavior, and release processes stable
across Codex, Claude Code, GitHub, and npm.

## Scope

The policy applies to:

- root `README.md`
- active files under `docs/`
- package README files
- provider-generated skill source text
- release notes and GitHub workflow comments

## Exceptions

Korean may appear when it is part of:

- user-facing example input
- multilingual trigger examples
- preserved research notes
- archived historical documents
- runtime output intentionally localized for a Korean prompt

When a Korean example is needed in an active engineering document, keep the
contract explanation in English and make the example clearly marked as an
example.
