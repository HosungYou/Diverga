## Executive Summary

Diverga v7.0.0 introduces the **Memory System** - a comprehensive research context persistence layer that enables cross-session continuity, automatic checkpoint enforcement, and research decision audit trails. This release transforms Diverga from a session-bound assistant into a **persistent research companion** that remembers decisions, maintains context, and guides researchers through the complete research lifecycle.

### 3. Decision Audit Trail

All research decisions are:

- **Immutable**: Never modified after creation (amendments create new entries)
- **Versioned**: Each amendment references original with `amends` field
- **Contextual**: Capture research question and prior decisions at time of decision
- **Exportable**: YAML format for transparency and reproducibility

```yaml
# Example decision entry (.research/decision-log.yaml)
- id: "dec-003"
  timestamp: "2026-02-04T14:30:00Z"
  checkpoint: "CP_RESEARCH_DIRECTION"
  stage: "foundation"
  agent: "diverga:a1"
  decision:
    selected: "Meta-analysis"
    alternatives:
      - option: "Systematic Review"
        t_score: 0.75
        rationale: "Good for narrative synthesis"
```

## Architecture Comparison: Diverga vs OpenClaw

### Overview

| Aspect | Diverga Memory v7.0 | OpenClaw Memory |
|--------|---------------------|-----------------|
| **Focus** | Research workflow | General assistant |
| **Scope** | Project-based | User-based |
| **Philosophy** | Decision audit trail | Conversation context |
| **Storage** | Structured YAML | Markdown files |
| **Search** | Keyword + semantic | Hybrid (FTS5 + vector) |

### Detailed Comparison

#### 1. Memory Scope & Purpose

| Feature | Diverga | OpenClaw |
|---------|---------|----------|
| **Primary Use Case** | Academic research lifecycle | Personal assistant tasks |
| **Memory Scope** | Per-project (`.research/`) | Per-user (`~/.openclaw/workspace`) |
| **Context Type** | Research decisions, checkpoints, stages | Conversations, facts, preferences |
| **Domain Specificity** | 44 specialized research agents | General-purpose agents |

**Diverga's Advantage**: Research-specific memory structure with paradigm awareness (quantitative/qualitative/mixed), VS methodology T-Scores, and PRISMA 2020 compliance.

**OpenClaw's Advantage**: Broader applicability for general tasks, simpler setup for non-research use cases.

#### 2. Storage Architecture

| Feature | Diverga | OpenClaw |
|---------|---------|----------|
| **Primary Format** | Structured YAML | Plain Markdown |
| **Structure** | Dual-tree (baselines/changes) | Flat (memory.md + daily logs) |
| **Schema** | Defined schemas per research type | Schema-free |
| **Versioning** | Immutable decision log with amendments | File-level versioning |

**Diverga Structure**:
```
.research/
â”œâ”€â”€ baselines/           # Stable foundations

```

**OpenClaw Structure**:
```

â””â”€â”€ bank/               # Typed memory pages
    â”œâ”€â”€ world.md
    â”œâ”€â”€ experience.md
    â””â”€â”€ opinions.md
```

**Key Difference**: Diverga's dual-tree separates verified research foundations from active work, enabling stage-based archiving and rollback. OpenClaw's flat structure prioritizes simplicity and human readability.

### Diverga's Unique Value Proposition

1. **Research-Specific Design**: Built for academic research workflows, not adapted from general-purpose systems
2. **Checkpoint Enforcement**: 17 defined checkpoints ensure human oversight at critical decisions
3. **VS Methodology Integration**: T-Scores prevent mode collapse in research recommendations
4. **Decision Audit Trail**: Immutable log satisfies research reproducibility requirements
5. **Paradigm Awareness**: Adapts behavior based on quantitative/qualitative/mixed methods
6. **44 Specialized Agents**: Domain expertise vs. general capabilities
7. **Bilingual Support**: Native Korean language support for Asian researchers

### OpenClaw's Unique Value Proposition

1. **Universal Memory**: Works across all types of tasks and conversations
2. **Hybrid Search**: Sophisticated FTS5 + vector retrieval
3. **Platform Integration**: WhatsApp, Telegram, Slack, Discord support
4. **File-First Philosophy**: Human-readable Markdown files
5. **Auto Memory Flush**: Intelligent context preservation before compaction
6. **Session Indexing**: LLM-generated slugs for searchable history
7. **Simpler Setup**: No domain-specific configuration required

```python
from lib.memory.src.memory_api import MemoryAPI
from pathlib import Path

memory = MemoryAPI(project_root=Path("."))
```
